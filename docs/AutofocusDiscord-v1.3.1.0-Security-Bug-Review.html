2. Security Review

Result: No medium, high, or critical issues found.

AreaVerdictNotes
Webhook URL / SSRF Pass WebhookUrlValidator enforces HTTPS, Discord hosts, and /api/.../webhooks/{id}/{token} path shape on every post path.
Secret exposure Pass Webhook URL/token not logged. Logs use sanitized filenames and outcome labels only.
Path traversal Pass Fixed AutoFocus folder; GetFullPath + prefix check; IncludeSubdirectories = false; *.json only.
Discord injection Pass Role pings validated (17–20 digit ID). Content uses Safe() for @everyone/@here. New live-failure embeds sanitized.
AF JSON parsing Pass 2 MiB max file size; 500 measure points max; JObject.Parse with typed accessors; no unsafe deserialization.
Network / TLS Pass Default HttpClient; 60s timeout; no certificate bypass added in v1.3.1.0.

2.1 Low-severity / defense-in-depth (optional, not blockers)

  • Embed titles use raw Filter from JSON without Safe(). A crafted local AF file could put @everyone in an embed title; requires write access to the user’s own AutoFocus folder and only affects that user’s webhook.
  • TOCTOU: path containment is checked before file settle/read; re-check immediately before read would tighten hygiene on shared machines.

3. Bug & Error Findings (v1.3.1.0)

SeverityLocationFindingStatus
High SequenceDigestCoordinator.cs Sequence digest race on back-to-back runs. OnSequenceFinished is async; PostSequenceDigestAsync re-read reports at post time. If the next sequence starts during wait/post, BeginSequence() clears sequenceReports — digest can be empty, wrong, or throw. Fixed in 1.3.1.1
Medium AutofocusLiveFailureMonitor.cs Live AF failure alerts blocked in digest-only mode. Posting required PostPerRun || PostFailures even when Post when autofocus ends without a JSON report was enabled. Fixed in 1.3.1.1
Medium SequenceDigestCoordinator.cs Pending post wait skipped. Reports landing after wait-loop timeout but before final count check could skip WaitForPendingPostsAsync, causing digest before per-run post completes. Fixed in 1.3.1.1
Medium Options.xaml / AutofocusGraphs.cs DiscordEnabled not synced with UI toggle. Master checkbox bound to Enabled only; if DiscordEnabled was off, status showed “Discord posting is turned off” with no way to fix from UI. Fixed in 1.3.1.1
Low SessionDigestService.cs Shutdown digest required Enabled. Disabling monitoring before NINA exit skipped configured shutdown digest despite collected reports. Fixed in 1.3.1.1
Low AutofocusRunTracker.cs lastReportUtc read outside lock. Theoretical race in live failure detection on tight timing; low impact on single-user desktop. Fixed in 1.3.1.1

4. Fixes Applied (v1.3.1.1)

FixImplementation
Sequence digest snapshot WaitAndSnapshotSequenceReportsAsync returns a frozen report list; PostSequenceDigestAsync(reports) posts from snapshot.
Pending post wait Always wait for pending per-run posts whenever sequence reports exist, regardless of how the wait loop exited.
Live AF in digest-only Removed gate requiring per-run/failure toggles; live failure posts only need PostLiveAfFailures, DiscordEnabled, and valid webhook.
UI / DiscordEnabled sync Master Enabled setter now sets DiscordEnabled to the same value and refreshes status.
Shutdown digest New PostShutdownDigestAsync bypasses Enabled check; teardown uses it when PostDigestOnShutdown is on.
Thread safety HasNewReportSince reads lastReportUtc under the tracker lock.

5. Build & Compile

Release configuration builds successfully with dotnet build -c Release. Release artifact contains 7 managed/native DLLs per publishing docs. NINA must be closed during deploy if post-build copy targets the plugin folder (DLL lock).
